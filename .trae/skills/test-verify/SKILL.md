# 测试/验证 Skill

## 目标

- 将验收标准转为可执行的测试用例与自动化覆盖
- 通过最小代价获得对发布质量的高置信度

## 协作约束

- 除非用户明确要求，不额外生成文档
- 不随意删除既有文档和代码（除非用户要求）

## 适用场景（触发条件）

- 新功能交付前需要验收与回归
- Bug复现、定位、修复验证
- 发布前的冒烟、接口兼容、权限与配额核验

## 测试层次建议

- 单元测试：纯逻辑与边界条件
- API集成测试：鉴权、错误码、分页与数据一致性
- 前端组件/逻辑测试：Pinia与关键交互逻辑
- 端到端冒烟：登录、对话、知识库关键路径

## 项目测试入口速览

- 后端：backend/tests/ 与 backend/test_*.py（pytest）
- 前端：frontend 的脚本包含 vitest（npm run test）

## 用例设计清单

- 权限：未登录、普通用户、管理员、资源归属
- 输入：必填/可选、长度、非法字符、越界、空列表
- 并发与幂等：重复提交、网络抖动、重试
- 资源：分页、排序、过滤、空结果
- 安全：注入、越权、敏感信息回显

## 验证输出（建议）

- 复现步骤与期望/实际对比
- 覆盖到的自动化用例列表
- 回归范围与未覆盖风险

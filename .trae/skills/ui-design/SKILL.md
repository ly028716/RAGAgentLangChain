# UI Skill

## 目标

- 让交互可理解、可操作、可反馈，且与现有风格一致
- 给前端提供可直接实现的结构、状态与视觉规则

## 协作约束

- 除非用户明确要求，不额外生成文档
- 不随意删除既有文档和代码（除非用户要求）

## 适用场景（触发条件）

- 新页面/新流程设计与改版
- 交互体验优化（加载、错误、空态、引导）
- 需要统一视觉规范与组件复用策略

## 项目UI上下文

- 组件库：Element Plus
- 全局样式：frontend/src/styles/global.scss 与 variables.scss
- 现有插画/图：frontend/images/

## 交付物（建议）

- 页面信息架构：模块划分与优先级
- 关键状态：默认/加载/空态/错误/成功/禁用
- 交互说明：点击、输入、校验、提示、撤销、快捷键（如适用）
- 视觉规范：间距、字号、颜色、图标使用原则（沿用现有变量）

## 设计自检清单

- 每个关键操作都有明确反馈与可恢复路径
- 文案可读、可定位，错误提示可行动
- 适配不同窗口尺寸（尤其对话与列表类页面）
- 表单校验规则与后端一致
